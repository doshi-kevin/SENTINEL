## ğŸ“ Your Data Files

### 1. **events.csv** - System Actions (Most Important!)
**What it contains:** Every action performed on the Windows system
- Reading files
- Writing data
- Executing programs
- Network communications
- Registry modifications

**Key Columns:**
- `timestamp` - When the action happened
- `event_type` - What kind of action (EVENT_READ, EVENT_WRITE, EVENT_EXECUTE, etc.)
- `subject` - UUID of the process that did the action
- `predicate_object` - UUID of what was affected (file, network, etc.)
- `size` - Size of data involved

**How to use it:**
- Sort by timestamp to see sequence of events
- Filter by event_type to find specific actions
- Look for unusual patterns around attack times
- This is your primary data for detecting attacks!

---

### 2. **subjects.csv** - Processes/Programs
**What it contains:** Every process (running program) on the system
- chrome.exe, firefox.exe, powershell.exe, etc.
- Parent-child relationships between processes
- When each process started

**Key Columns:**
- `uuid` - Unique ID for this process
- `cid` - Process ID (PID)
- `cmd_line` - Full command that started the process
- `parent_subject` - UUID of parent process
- `privilege_level` - Admin rights or normal user

**How to use it:**
- Match UUIDs with events.csv to see what each process did
- Follow parent-child chains to see how processes spawned
- Look for suspicious process names or command lines
- Check privilege escalation (normal user â†’ admin)

### 3. **network.csv** - Network Connections
**What it contains:** Every network connection made
- Outbound connections to servers
- Inbound connections
- Ports used
- Protocols (TCP, UDP)

**Key Columns:**
- `uuid` - Unique ID for this connection
- `local_address` & `local_port` - Your computer's end
- `remote_address` & `remote_port` - Server being connected to
- `ip_protocol` - Type of connection (6=TCP, 17=UDP)

**How to use it:**
- Match UUIDs with events.csv to see network activity
- Look for connections to suspicious IPs
- Check for unusual ports
- Find Command & Control (C2) server connections
